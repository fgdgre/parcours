# Parcours v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a static, phone-first Nuxt 4 PWA that plays the complete verified French A0 program (skeleton for all 7 chapters, deep content for Chapter 1) with SRS flashcards, text exercises, TTS dictation, and ASR speaking checks â€” progress in localStorage, deployable to Netlify.

**Architecture:** SPA (`ssr: false`) generated to static files. All course content is typed JSON under `app/content/` loaded via static imports; the app is a generic player keyed on stable lesson ids. Pinia store persists progress/SRS/settings to localStorage with a versioned schema and JSON export/import.

**Tech Stack:** Nuxt 4, TypeScript, Pinia, Vitest (logic tests only), plain scoped CSS (no UI framework), Web Speech API (TTS + ASR).

## Global Constraints (from spec â€” verbatim intent)

- Phone-first: layouts designed at 375â€“430 px; bottom tab bar; â‰¥44 px touch targets.
- External media embedded/linked, never copied. Every embed node keeps a fallback external URL.
- Content = data. Adding chapters touches JSON only. Progress keys on stable lesson ids that are never renamed.
- Flashcards FRâ†’EN only.
- localStorage schema carries `schemaVersion: 1`.
- Node 26 / npm 11 available. No backend, no accounts, no analytics.
